# Mine Road リメイク差分台帳（原作 vs v0.15.0）

チケット #119（2026-07-18）。原作挙動の正本 = `mineroad-original-spec.md`（以下 §n はその節番号）。リメイク側は `mineroad/web/app.js` / `tiles.js`（v0.15.0）の実コード抽出。

**分類**: 忠実 = 原作と同機構 / **翻案** = 意図的な引き直し（記録あり or 合理）/ **誤解由来** = 旧仕様まとめの誤りや推測が実装に入ったもの / 未実装 = 原作にあってリメイクに無い
**深刻度**: 高 = 手触りの核が別物（修正発注対象）/ 中 = 遊びの質に効く / 低 = 細部・意図的サブセット

## 修正発注キュー（深刻度順の推奨）

1. **水/マグマの機構総取り替え**（誤解由来・高）: 固定オーバーレイ + SP×3/×4 → 流動セルオートマトン + 「SWIM 値ターンまで無傷→超過後 HP 直撃」+ 浮力。#119 発注枠 ②
2. **埋没モンスターの機構替え**（誤解由来・高）: 掘削時抽選スポーン → 生成時埋め込み + 毎ターン自力脱出抽選。#119 発注枠 ③
3. ~~**ランタイムスポーン導入**（未実装・中）: 行動ごと 30% で 16〜28 マス圏に追加湧き。「掘り尽くしたら安全」を消す原作の緊張源~~ → **v0.19.0 で消化（#124）**
4. **なだれの機構替え**（誤解由来寄り・中）: ワンショット 8 ダメージ + 即時列落下 → 3 段階弱化 + 1 マス崩落 + 連鎖伝播 + 埋没継続ダメージ
5. **女の子ロストの重み**（翻案・中）: 元位置復帰（実質ノーリスク）→ 原作は完全消滅。少なくとも中間（そのダイブ中は再救出不可等）を検討
6. 低群: SP 経済の再調整（不足分 1/2 変換・地表無料・掘削コスト式）、攻撃の乱数幅、モンスター特殊個体、落下 1 マス/ターン、等

## 総括

- **誤解由来の中核 2 件（水/マグマ・埋没モンスター）が最優先**。いずれも旧仕様まとめの推測記述（§9 の誤り 2〜4）がそのまま実装に写った系譜で、v0.15.0 の掘削と同型の「原作合わせ」増分として発注可能
- 翻案群の大半は STATUS.md に判断記録があり健全（決定論方針・スマホ片手 UI・規模縮小）。記録なき翻案で手触り差が大きいのは**女の子ロストの軽さ**と**復活時のダンジョンリセット**
- 原作の緊張源として未輸入だった**ランタイムスポーン**（湧き続ける地下）と**埋没の継続ダメージ**（埋まる恐怖）は v0.19.0（#124）/ v0.17.0（#121）でそれぞれ消化済み
