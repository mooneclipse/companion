## §1 リソース（HP/SP の二段ゲージ）

- 初期値: **最大 HP=100 / 最大 SP=1000**（`cm.java:650-653`）。DIG/ATK 初期 1
- **SP 不足時の HP 変換**: `cm.h(int)`（`cm.java:1216-1222`）= SP から引き、**負になった不足分の 1/2（整数除算）を HP から引く**。「SP0 で即 HP 全額」ではない
- 行動別 SP コスト（一次確認済み）:
  | 行動 | コスト | 根拠 |
  |---|---|---|
  | 移動（y>5 の地下のみ、落下含む座標変化全て） | 2/ターン | `cm.m()` `cm.java:1276-1279` |
  | 移動（地表帯 y≦5） | 0 | 同上の条件分岐 |
  | 掘削 1 回 | `clamp(8 − DIG/2, 1, 8)` | `cm.java:866-871` |
  | 鉱石オブジェクト押し出し | 5（HP 変換なしの直接減算） | `cm.java:284-292, 1282-1284` |
  | その場タップ | 1 | `cm.java:1272-1274` |
  | 攻撃 | **0**（SP 消費呼び出しなし） | `cm.java:1041-1044` |
- 全回復 `cm.k()`（`cm.java:1255-1263`）: HP/SP を最大値へ、状態異常(36)解除。地上到達時と力尽き復活時で共用

## §2 移動・ジャンプ・重力

- 1 タップの解決優先順位（水平 `cm.i()` / 下 `cm.h(n)` / 上 `cm.k(n)` で共通構造）: ①移動先が歩行可(`n.w()`)→移動 ②押せる鉱石→押す(SP5) ③敵→攻撃（**移動しない**） ④掘削可(`n.ae()`)→掘る（**移動しない**。全方向、掘るだけ。空いた先へは次の入力）
- **重力**: `n.n(x,y)`（`n.java:2386-2388`）=「はしご/水/マグマ上でなく、真下が歩行可なら落下」。落下は**毎ターン 1 マスずつ**（モード 3 継続、`cm.java:334-347`）。**落下ダメージは未発見**（？存在しない可能性が高い）
- **ジャンプ**: 高さ固定 1（`cm.c[11]=1`、`cm.java:630-632`）。上/斜め上入力で 1 マス上がってその場で終了、以後は通常重力。真上入力は横オフセット強制 0（`cm.java:780-782`）
- **はしご**（オブジェクト ID 7,8,20,22,26、`n.java:1834-1845`）: 効果は**重力無効化のみ**。上下移動は通常の上下入力をそのまま使う。プレイヤーによる設置手段は ？（アイテム種別スイッチに見当たらず）
- **水/マグマ中も重力無効**（`n.y()` が水/マグマを含む）= 浮力。垂直移動は自力入力

## §3 掘削（v0.15.0 裏取り済み分の再掲 + 補強）

- タップ 8 方向（`bc.java:210-244`）。真上掘りは power ゲートのみで無条件（`cm.java:349-407`）。斜めは横隣 or 縦隣空間が前提。掘削と移動は別アクション（掘り抜きで前進しない。§2 ①〜④の優先順位どおり**全方向共通**）
- ブロック耐久はファミリー制（`ck.java:7`）: SOIL_HARD(56-58) もなだれ土(72-74) も**総耐久 12 打撃・3 段階**。掘削力(DIG)は SP コスト減（§1）と与ダメージに効く
- 掘削エンジンは `n.h()` → `a(int,int,int,true)`（`n.java:2250`）。モンスターの土破壊（§5）・水食（§6）も同一エンジン

## §4 水/マグマ（流動 + 生存ルール）

- **流動**: 毎ターンのセルオートマトン（`n.java:751` P() → `n.java:1851` b()）。左右下へ流れ、空マスへ最低密度 24/32 を生成、同系合流で密度+1、31/39 で満水（v0.15.0 裏取り済み）。P() の駆動元・走査範囲は ？（jadx 復元欠落）
- **水中の消耗は HP 直撃・SP ではない**: `cm.e(n)`（`cm.java:1146-1169`）= 水中滞在ターン数カウンタ(c[34])を毎ターン +1、**SWIM 値(c[33]) を超えるまで無ダメージ**。超過後は毎ターン **HP − clamp(4 − SWIM/5, 1, 4)**。水から出るとカウンタ 0 リセット（`cm.java:1082-1084`）= 息継ぎ
- 水中の泡オーバーレイ（`g()==16`）を踏むとカウンタリセット + 泡消費（`cm.java:1149-1152`）= 空気ポケット
- **マグマ**: 毎ターン **HP − 最大HP/5**（`cm.i()` `cm.java:1224-1226`、死因コード 2）
- 移動コスト側の割増は無し（§1 の一律 SP2 のみ）。SWIM の効きは「無傷滞在ターン数の延長 + 超過後ダメージの減額」であり、乗算軽減ではない

## §5 モンスター

- **スポーン 2 系統**:

## §6 なだれ/落盤（+ 埋没ダメージ）

- なだれ土 = ID **72/73/74** の 3 段階（dungeon.csv `SOIL_AVALANCHE_HARD`=74）。生成時は帯別重み（裏庭 0%、埋没城跡 深部で最大 12%）
- **崩落機構**（`n.java:918-949`、一次確認済み・逐語）:
  1. **真下の固体支持が失われる**（`ad(x,y+1)` false = 下が空間/水/マグマ）と、毎ターン 1 段階ずつ弱化（74→73→72 = 2 ターンの「軋み」猶予）
  2. 最弱 72 でなお支持なし → 自マスを空(0)にし**真下へ 72 を生成**（1 マス落下で下を塞ぐ）
  3. **連鎖伝播**: 真上の土は無条件、左右の土は各 30% で 72 化 → 次ターン以降の連鎖崩落（上方向へ陥没が広がる）
- **崩落コード自体にダメージ処理はない**。実体は §0 の埋没: 崩落で自マスが土になったユニットは「土の中に居る」状態になり、プレイヤーは**毎ターン HP − 最大HP/10**（`cm.h()` `cm.java:1206-1213`、死因コード 4、一次確認済み）、モンスター/女の子は毎ターン HP−1（bk.e）。掘って出るまで継続
- **水食による自然発生**（`n.java:951-956`、一次確認済み）: 土マスは毎ターン、基礎 0.1% × 周囲 8 方向の水タイル数×10% の確率で 1 打撃の浸食を受け、削り切れたら**なだれ土 74 に変質**（水辺の土は時間経過で崩れやすくなる）
- 硬岩(96+)/古代壁/石柱は `af()` 範囲外で崩落に巻き込まれない。石柱等の独自落下物理は無し（P() の default 分岐で確認）
- ？: なだれ土の視覚区別可否（描画コード未発見）/ P() の駆動元
