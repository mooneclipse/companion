## 3. 横断的な留意点

- **app.js 1322 行モノリス**: 現状は健全に増分できているため強制分割はしない。モンスター/商人/育成で状態が増えて 1 ファイルが重くなったら、責務分割（state / world / mechanics / entities / economy / render / input / ui）を**そのとき**検討する（早すぎる再設計はしない）。

## 4. 進め方のリズム

- 1 セッション 1 増分。着手時に該当チケットを「着手中」へ、STATUS に翻案判断を先に記録。
- 実装 → playtester → code-reviewer → commit（既存ログ調、Co-Authored-By なし、push なし＝(C) ローカル git）→ OWNER 実機 → 承認で本番反映 → STATUS に結果追記。
- 不可逆操作（本番反映）は OWNER 承認後。設計の 2 周目（条件分岐/閾値の積み増し）は上位 `~/companion/CLAUDE.md` のルールで一段引く。
