## 2. 残りの増分バックログ（1 増分 = 1 チケット、OWNER 発注順）

原作の全システム（9 ダンジョン / 45 アイテム / 20 モンスター / クラフト / 商人 / 仲間）へ忠実に近づける。**次の本命は OWNER 明示でモンスター**。

1. **モンスター + ハザード**（§9 全20種 / 水・マグマ・なだれ・落盤 §6/§8）= **死の緊張を設計から出す本命**。敵スポーン（決定論）・戦闘（攻撃/防御/EXP）・GIRLATK（護衛中の女の子が狙われる）・埋没掘り出現率。数値だけでなく危険コンテンツで撤退判断を重くする。
   - **【モンスター部分 = v0.5.0 で実装済み（2026-06-20）】**: 浅層向け忠実サブセット6種（BAT/SLIME/SLIME HALF/SNAKE/WORM/SPIDER）・bump-to-attack 戦闘・空間/埋没掘り2系統の決定論スポーン・GIRLATK・EXP/ドロップ verbatim。**力尽き率 0/20→5/20(25%)＝死の緊張を敵コンテンツで出した数値裏取り**（reckless 死亡/optimal 生存、資源数値の単独いじりなし）。詳細は STATUS「v0.5.0」エントリ。本番反映は OWNER 承認待ち。
   - **【ハザード部分 = 水/マグマ = v0.6.0 で実装済み（2026-06-21）】**: 浸水サブシステム（原作「水中・マグマ中は泳げるがスタミナ激消耗・マグマ特に危険・SWIM で軽減」）を、tileType/girlPositions/oreAt/monster 非介入の決定論オーバーレイ `hazardAt(col,row,seed)`（別位相 hash3）で実装。水=SP×3 / マグマ=SP×4＋HP chip 2、二段ゲージ（spendAction の SP→HP / takeDamage）へ接続。深度ゲート＝水 row≥5・マグマ row≥9。SWIM 軽減係数は `hazardSpMult`/`hazardHpChip` の 1 点に切り出し（育成増分フック、対症療法回避）。playtester 新ゲート W ALL PASS（3回連続決定論緑）・非介入 verbatim。詳細は STATUS「v0.6.0」エントリ。本番反映は OWNER 承認待ち。**力尽き率 25%→25% 不変**＝固定 seed 裏庭が浅くマグマ深層が支配する前に既存敵被弾で死ぬ盤面のため（メカは W3/W4 で直接実証済み・資源数値の単独いじりなし）、浸水の緊張が支配的に効くのは深ダンジョン増分での発露が本筋。
   - **【ハザード部分 = なだれ/落盤 = 次増分送り】**: なだれ土・落盤は別タイル物理サブシステム（崩落）で、浸水（水/マグマ）とは独立。全20種ロスター・ボス級（SOIL DRAGON/BEAR）も次以降。

## 3. 横断的な留意点

- **app.js 1322 行モノリス**: 現状は健全に増分できているため強制分割はしない。モンスター/商人/育成で状態が増えて 1 ファイルが重くなったら、責務分割（state / world / mechanics / entities / economy / render / input / ui）を**そのとき**検討する（早すぎる再設計はしない）。
- **決定論**: ランタイム乱数（Math.random / Date.now）禁止を厳守。新システムも seed 由来ハッシュ（既存 `hash3`/`oreAt` 流儀）で決定論に。
- **検証ゲート**: 各増分で playtester（`tests/debug-mineroad.mjs`）ALL PASS + 既存作 URL 不変回帰 + 短高 viewport はみ出し0。本番47825 非接触（別ポートで検証・PID 直 kill）。
- **配信ゲート**: 本番反映（`systemctl --user restart companion-games`）は**各増分で OWNER 実機承認を取ってから**。コミットは配信ゲートを越えない。

## 4. 進め方のリズム

- 1 セッション 1 増分。着手時に該当チケットを「着手中」へ、STATUS に翻案判断を先に記録。
- 実装 → playtester → code-reviewer → commit（既存ログ調、Co-Authored-By なし、push なし＝(C) ローカル git）→ OWNER 実機 → 承認で本番反映 → STATUS に結果追記。
- 不可逆操作（本番反映）は OWNER 承認後。設計の 2 周目（条件分岐/閾値の積み増し）は上位 `~/companion/CLAUDE.md` のルールで一段引く。
